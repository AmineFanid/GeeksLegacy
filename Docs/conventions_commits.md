# Commits Convention

Notre convention pour les commits sur **github**


## Type :

- **feat** : Nouvelle fonctionnalité
- **fix** : Correction d'un *bug*
- **docs** : Ajout d'un document
- **style** : Modification ou Ajout de style ou design (aspect graphique)
- **refactor** : Modification de code
- **test** : Ajout, modification ou suppression de test 
- **perf** : Modification de code axé sur l'amélioration de la performance
